Couchbase vs MySQL — Full Benchmark Report

Load test comparison across Query & Consume endpoints · Caller-side + Backend (Grafana) metrics · 25 Feb 2026

Introduction

This report compares Couchbase and MySQL as the backing datastore for the Product Variants Reader service, under two distinct workload scenarios:

  • Query scenario — read-only load against the query endpoint, simulating catalog read traffic.
  • Consume scenario — combined writes and queries via a consume endpoint, simulating real-time event processing alongside reads.

Each scenario was tested at two scale levels: 1 pod (single-instance baseline) and 10 pods (horizontal scale).

Environment

  • App pods: 4 CPUs, 6 GB memory each (JVM max heap: 3 GB)
  • Couchbase & MySQL: identical hardware, both configured with replication to another cluster
  • Dataset: 150K records in each datastore — the goal is to compare application-level capabilities, not to stress the databases themselves
  • Metrics: caller-side (load test client) throughput & latency + backend (Grafana) P95/P50 response times & CPU usage
